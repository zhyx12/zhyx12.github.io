Pytorch 0.4迁移指南

发表于 2018-09-11 | 更新于 2018-11-15 |

版本迁移

0.4最大的改变在于将Tensor和Variable合并。本文的目的在于厘清一些之前比较模糊的概念，如求导机制中的requires_grad，volatile，leaf，non-leaf等，并介绍新旧版本的差异。

自动求导

requires_grad是自动求导的核心标志。在loss.backward()时，遇到一个tensor的require_grad=True，就会为它计算误差，并且保存（累加）在grad属性里，如果require_grad=False，就不为该tensor计算误差。 可以通过在每一次前向计算时设置模型参数的requires_grad=False，使得loss.backward()时这些网络参数的梯度（grad）为None。

网络权重的梯度的累加和optimizer中是否有该权重是相互独立的。

optimizer只需要创建的时候参数的requires_grad=True就行，step()时=False也没有关系，grad属性可能是None，这时调用optimizer.step()就不会改变参数的值。requires_grad等于False并不代表grad一定是None，可能只是临时设置为False，但是有之前累积的grad。

因为联系整个计算图的是中间的计算结果，网络参数本身可以看做是叶子节点，所以才可以做到网络参数的部分更新，这时一定要保证

旧版本

之前版本 Variable的属性：

  • data
  • grad
  • requires_grad
  • volatile

  • data

和optimizer的关系

  1. 错误信息：optimizing a parameter that doesn't require gradients。送到优化器中的参数需要requires_grad = True
  2. 错误信息：can't optimize a non-leaf Variable 送到优化器中的参数只能是leaf或者网络的权重参数，不能是中间的输出变量，如特征图（特征图一般也不需要优化，只需要传递误差就行）

leaf node指的是用户自己定义的变量，以及模型的参数，而中间的结果都是non-leaf node。

detach() 函数返回一个新的Variable，从当前的计算图中脱离，不需要计算梯度。返回的Variable和原来的Variable共用一个tensor数据，in-place的操作会彼此影响。

register_hook(hook)注册一个反向传播的hook,hook是一个函数，输入是正常的grad,输出是新的grad或者None hook(grad) -> Tensor or None

detach_() 从计算图中脱离，成为一个叶子节点。detach_()之后required_grad自动变成False

新版本

Variable的属性 - data 保留，不变 - grad 保留 - requires_grad 保留 - volatile 废弃 - is_leaf 保留 - grad_fn 保留

volatile去掉了，改成了使用上下文管理器来设置是否需要计算grad

.data属性和detech()

在旧版本里.data属性获取的是tensor，就和计算图没有关系了，同时共享了底层的数据，如果对tensor进行in-place的操作，也会对梯度的反向传播产生影响（该变量之前的变量的梯度计算会出错），但是不会报错。 0.4版本中.data属性保持了同样的含义，会返回一个共享数据的Tensor,和计算历史无关，同时requires_grad = False，如果进行了in-place的操作，效果同上。

推荐的方式是使用detach()，返回一个新的Variable或者tensor。如果detach()之后的量进行了in-place操作，会报错，但是抱错的位置不一样。0.3.1更早一点。

tensor

torch.Tensor创建函数是默认的tensor类型的别名，torch.Tensor()如果接受多个数字的话，会把他解析成size，如果是序列的话，会解析成数据，这些在新旧版本都一样的，可以参考0.3.1Tensor文档中对于Tensor()创建函数中不同参数的定义。

之前的版本也存在0维的tensor，torch.Tensor()创建的就是0维的tensor，但是这个没有什么用途（？），而且和0.4.0的不一样，0.4.0的zero-dimensional tensor指的是仅包含一个元素，dim()=0，size()=torch.Size([])的tensor，0.4.0中，torch.Tensor()虽然也会返回一个不包含任何元素的tensor，但是dim()=1，size()=torch.Size([1])。

0.4.0增加了torch.tensor(data, dtype=None, device=None, requires_grad=False)的tensor创建函数，在之前的版本中，torch.tensor的类型是class 'module'，0.4.0中的类型是builtin_function_or_method。torch.tensor传入的第一个参数总是数据，包括Python number，list, tuple等，而且会拷贝原来的数据。

numpy torch 类型 接受参数
np.array torch.tensor builtin_function_or_method data
np.ndarray torch.Tensor type np.ndarray只接受shape，torch.Tensor接受data或shape(单纯的数字)

0.4.0重新定义了zero-dimensional tensor，即用tensor.dim()函数查看的话，返回的数字是0。创建的话可以使用torch.tensor(single_number)，

item()可以用在只包含一个元素的tensor上，zero-dimensional tensor或者dim()=1，torch.size()=torch.Size([1])的都可以
